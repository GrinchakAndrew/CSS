    CSS Box Model, cont.: Margins and Padding
    • The 'Padding': if you use the short-hand notations:
      - the first bullet shows application of the padding this way: top, right, bottom, left; - the second bullet shows application of the padding this way: top, left AND right sides, the third value applies to the bottom; - if you only use two values, the first value applies to the TOP and the BOTTOM, while the second applies to both LEFT and RIGHT sides; - if you simply use the single value, it will be used for all sides equally.
    • The 'Padding': important considerations:
      - Block elements will always will out ALL of the available space, if they are allowed;
    • The 'Padding': important considerations:
      - By adding padding to the element with undefined width, it will most likely grow in height because the padding will cause the content area to shrink;
    • The 'Padding': important considerations:
      - If the child element, whose display style is block, is assigned 100% width, with the padding, it will actually overflow its parent, because the padding is added OUTISDE of the element's width; - this is possible to change by defining the 'box-sizing' property.
