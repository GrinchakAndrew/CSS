    CSS Layout-1
    • The 'Layout': CSS Positioning Schemes: - Normal Flow: default positioning, elements are stacked in the order of the original HTML layout.
    • The 'Layout': CSS Positioning Schemes:
    • The 'Static': CSS positioning: is the default layout scheme.
    • The 'Relative': CSS positioning: serves for offsets, leaving the elements where is originally belongs:
    • The 'Absolute': CSS positioning: removes the elements from the normal flow of the document:
    • The 'Absolute': CSS positioning: looks for the positioning of element's parent, else for the viewport [the body]:
    • The 'Absolute': CSS positioning: when the parent has positioning set:
    • The 'Fixed': CSS positioning: always against the position of the active viewport:
    • The 'Z-index': CSS positioning: controlled layering, which element is on top of the others.
      By default positioned elements will appear on top of non-positioned elements, if those two overlap.
    • The 'Inline' Elements:
      - are stacked up one after another, and word-break is after the lines ends;
      - may contain text or other inline elements;
      - width and height are not applicable;
      - before and after there is no default word-break;
      - size depends on the contents of the inline element;
    • The 'Inline' Elements:
      - < a >
      - < b >, < strong >
      - < br >
      - < i >,< em >
      - < img >
      - < span >
      - < sub >
    • The 'Block' Elements:
      - take up all the height, width available;
      - start from the new line;
      - possible to nest inside other block elements;
      - have the Box Model dimensions;
    • The 'Inline' Elements:
      - < div >
      - < h1 >, < h2> ...
      - < ol > , < ul > , < dl >
      - < p >
      - < table >
      - < form >
