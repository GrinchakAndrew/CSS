    Width / Height of 100%
    • Can I use width = 100% to make my page take up all of the height of the screen?
    • Actually, no: 'height: 100%;' is equal to the height of the parent element. Height of 100% does not stand for the 'height of the whole window':)
    • Why is that? The parent of your container is the 'body', and its height is set to 'auto' by default; This means, its height is equal to that of the content. Of course, you can add 'height: 100%;' to the 'body', however, this is not enough either.
    • Why is that? Because the parent of the 'body' is the element 'html', whose 'height' property is set to 'auto' and it will also expand to take up the whole width of the content. And now, if you will add 'height: 100%;' to the 'html' element also, everything will work out.
    • The root element to the 'html' is the 'viewport'. Viewport is the browser window. So, if you set 'height: 100%;' to 'html', it is just the same as saying: get the browser 100% height:
      html, body, .container {height: 100%;}
    • What if the parent element has property 'min-height', and not the 'height'?
    • Workaround example in code JSFiddle: .parent { min-height: 300px; height: 1px; /* Required to make the child 100% of the min-height */ } .child { height: 100%;}
    • By the styling rule 'width: 100%;', we want our element to take up the whole of the width of the parent's.
    • The 'width' property is NOT the absolute dimensions of the element, rather the size of its inner content, which is a big difference.
    • If you add 'padding' and/or 'border' to the element with 'width: 100%;', it will no longer fit the parent: JSFiddle
    • There are several possible workarounds. The first and the best one is avoiding the property 'width: 100%;', as in this case it's useless. If the sibling element is a block-element, it will take up the whole of its parent's width automatically(no problem with paddings or borders). But if we are working with 'inline-block' element, we are up against another problem.
    • As alternative, you can use the 'calc()' property to calculate width: 'width: calc(100% â€” 3em);'. However, it will not work either. First off, we will need to calculate the 'padding' + 'border'. Secondly, calc() has poor browser support(does not work in IE 8, Safari 5, Opera 12).
    • The final and best idea is usage of the 'box-sizing: border-box;' property. It will account for 'padding' and 'border' of inner content. 'box-sizing' has good browser support (IE8+, Opera 7+).
    • Conclusion: do not use the 'width: 100%;' property withouth 'box-sizing: border-box;'.
